## Design Context

### Users
MathBounty is a decentralized math bounty marketplace on Ethereum Sepolia. Users are crypto-native individuals with MetaMask wallets. Every wallet has equal permissions—anyone can be a poster or a solver. The context is self-directed, online, and trustless: users interact directly with escrowed ETH bounties without intermediaries. They are comfortable with blockchain concepts and are motivated by economic reward for intellectual effort.

### Brand Personality
- **Trustless** — No intermediaries; code is escrow and judge.
- **Permissionless** — No roles or gatekeepers; any wallet can participate.
- **Incentivized** — Everything is driven by economic reward for intellectual effort.

The emotional goals are: **focus** (reading and solving math), **urgency** (deadlines drive action), **confidence** (transparent escrow and deterministic payouts), and **excitement** (earning ETH rewards).

### Aesthetic Direction
**"Brutalist Academic"** — Raw, unapologetic, and structurally exposed. The visual language of concrete architecture meeting raw terminal output. No decoration, no softening, no gradients. Every element exposes its structure: borders are thick and visible, corners are sharp, typography is aggressively hierarchical. This is math as machinery — precise, cold, and beautiful in its rawness.

- **Theme**: Dark mode default (raw terminal black), with a stark light mode option. Dark mode evokes a CRT monitor in a basement lab; light mode evokes a photocopied zine.
- **Differentiation**: When users land on MathBounty, it should feel like they're interfacing with a piece of infrastructure — not a consumer app. The unforgettable element is the collision of rigorous mathematical content with raw, brutalist interface design that refuses to be "friendly."

**References**: Brutalist web design (raw HTML aesthetics, exposed structure), early computer terminals (CRT glow, monospace discipline), concrete architecture (raw materiality, no finish), photocopied academic zines (stark contrast, utilitarian layout).

**Anti-references**: Generic EdTech platforms, corporate SaaS dashboards, traditional tutoring sites, dark-crypto-neon UIs with glassmorphism and gradient text, "friendly" rounded-corner consumer apps.

### Design Principles
1. **Expose the Structure** — Borders, grids, and boxes are visible by design. Nothing is hidden behind subtle shadows or soft gradients.
2. **Typography as Weapon** — Display type is enormous and commanding. Body type is clean and subordinate. Hierarchy is created through scale and weight, not decoration.
3. **Zero Radius, Maximum Contrast** — Sharp corners everywhere. Stark black/white/contrast ratios. No softening.
4. **Code is Law, Design is Machine** — Every visual decision reinforces the mechanical, trustless nature of the platform. The interface should feel like it was built by engineers for engineers.
5. **Information Density is a Feature** — Users need to scan bounties, compare deadlines, and assess rewards quickly. Density is not clutter if hierarchy is clear.

### Typography
- **Display**: `Bebas Neue` — A condensed, all-caps display sans-serif with an aggressive, architectural presence. Evokes factory signage, sports jerseys, and raw industrial labeling. Used for headings, labels, and any text that demands attention.
- **Body**: `DM Sans` — A clean, geometric sans-serif with excellent legibility. Neutral but warm enough for reading. The mechanical counterpart to Bebas Neue's aggression.
- **Mono**: `JetBrains Mono` — A technical monospace designed for code. Clear distinction between similar characters, excellent for addresses and hashes.
- **Scale**: 5-step modular scale with fluid sizing (`clamp`) for marketing/pages, fixed `rem` for app UI. Minimum 1.25 ratio between steps.
- **Line length**: Cap body text at ~70ch for comfortable reading.

### Color Palette (Hex — Brutalist)
- **Brand Red**: `#ff3b00` — Aggressive, urgent, impossible to ignore. The color of warning labels and raw energy.
- **Dark Background**: `#0a0a0a` — Near-black, raw terminal darkness.
- **Light Background**: `#fafafa` — Stark white, photocopied paper.
- **Dark Text**: `#f0f0f0` — Off-white for dark mode, reducing eye strain while maintaining contrast.
- **Light Text**: `#0a0a0a` — Stark black for light mode.
- **Success**: `#00c853` — Harsh, terminal green.
- **Error**: `#ff1744` — Aggressive, alarm red.
- **Neutrals**: Pure grays with no warmth — `#888888`, `#555555`, `#333333`, `#222222`, `#111111`.

### Layout & Space
- 4pt spacing scale with semantic tokens (`--space-sm`, `--space-md`, etc.): 4, 8, 12, 16, 24, 32, 48, 64, 96.
- Use `gap` for sibling spacing, not margins.
- Vary spacing for hierarchy—tight groupings, generous separations.
- Self-adjusting grids: `repeat(auto-fit, minmax(280px, 1fr))` for bounty cards.
- Container queries for component-level responsiveness.
- Asymmetric layouts where appropriate; left-aligned text preferred.
- Exposed grid patterns and visible structural lines.

### Motion
- Focus on high-impact page-load orchestration with staggered reveals.
- Use exponential easing (`ease-out-quart/quint/expo`) for natural deceleration.
- Animate only `transform` and `opacity`.
- Respect `prefers-reduced-motion`.
- Occasional glitch effects for emphasis — but used sparingly.

### Absolute Bans
- No rounded corners (`border-radius: 0` everywhere).
- No side-stripe borders (`border-left`/`border-right` > 1px) on cards, lists, callouts, or alerts.
- No gradient text (`background-clip: text` with gradients).
- No glassmorphism or backdrop blur used decoratively.
- No generic card grids (icon + heading + text, repeated endlessly).
- No hero metric layout template (big number, small label, gradient accent).
- No soft drop shadows. If shadows exist, they must be harsh and architectural.
- No modals unless absolutely necessary.
- No "friendly" pastel colors or softening tints.
